# DocReview 智能体系统 - 产品需求文档

## 概述

- **摘要**：基于 LangGraph 框架的多智能体文档审查系统，自动审查产品需求文档（PRD）、技术方案等，确保文档质量
- **目的**：为开发团队提供高质量文档审查工具，在正式开发前发现问题、统一理解、降低返工
- **目标用户**：产品经理、技术架构师、开发工程师

## 目标

- 实现规格文档自动生成（根据用户任务描述）
- 执行严格的六步审查流程（核心闭环、一致性、需求原子化、技术可行性、风险、可执行性）
- 支持多轮迭代审查与自动修订
- 提供执行门禁机制（需用户确认后才开始实际开发）
- 集成 Sequential Thinking 和 Context7 MCP 服务提升审查质量

## 非目标（范围边界）

- 不支持多用户实时协作审查（单用户单实例模式）
- 不提供 Web UI 界面（仅 CLI 交互）
- 不支持多文档并行审查（v1.1 规划）
- 不提供 LLM 微调能力
- 不监听任何 HTTP 端口（v1.0 纯 CLI）

## 背景与上下文

现有开发流程中，文档质量问题常导致后期返工。本系统旨在通过 AI 智能体实现自动化文档预审查，在早期发现问题。

## 功能需求

### Supervisor Agent（主管智能体）
- **FR-S1**：接收用户通过 CLI 提交的任务/请求
- **FR-S2**：将用户任务分解为可执行子任务
- **FR-S3**：自动生成或更新结构化规格文档
- **FR-S4**：将规格文档提交给 DocReview 子智能体
- **FR-S5**：根据审查报告自动修订规格文档
- **FR-S6**：管理审查迭代循环，控制终止条件
- **FR-S7**：审查通过后中断并等待用户确认
- **FR-S8**：用户确认后开始实际开发任务执行
- **FR-S9**：维护审查轮次计数、问题状态、修订历史

### DocReview 子智能体
- **FR-D1**：核心闭环提取与验证
- **FR-D2**：术语、FR/AC、目标/约束等一致性检查
- **FR-D3**：模糊需求分解为原子需求，检查 NFR 覆盖
- **FR-D4**：基于技术栈模拟实施，验证依赖和决策
- **FR-D5**：风险检测与严重程度分类
- **FR-D6**：从开发者视角判断文档可执行性
- **FR-D7**：生成结构化 Markdown 审查报告
- **FR-D8**：识别修订版本文档，生成对比摘要
- **FR-D9**：严格独立性：仅发现问题和建议，不直接重写文档

### 阅读工具
- **FR-R1**：读取指定路径的文件内容
- **FR-R2**：在文档中按正则表达式搜索内容
- **FR-R3**：列出指定目录的文件和子目录
- **FR-R4**：生成两个版本文档的 unified diff
- **FR-R5**：限制文件访问在工作目录范围内（路径安全）

### 终端工具
- **FR-T1**：在系统终端中执行命令
- **FR-T2**：捕获 stdout 和 stderr
- **FR-T3**：返回命令执行的退出状态码
- **FR-T4**：支持命令执行超时设置
- **FR-T5**：命令失败时支持重试（含指数退避）
- **FR-T6**：危险命令白名单拦截机制

### 联网搜索工具
- **FR-W1**：根据查询关键词搜索相关网页
- **FR-W2**：获取指定 URL 的网页内容并转为 Markdown
- **FR-W3**：验证第三方 API/框架的可用性
- **FR-W4**：检测项目依赖的过时版本

### MCP 服务集成
- **FR-M1**：Sequential Thinking 多步推理
- **FR-M2**：Context7 长上下文检索
- **FR-M3**：标准 MCP JSON-RPC 协议通信

## 非功能需求

- **NFR1**：工具调用失败时自动重试（最多 3 次，指数退避 1s/2s/4s）
- **NFR2**：模块化设计，单一职责，>20 行代码时抽象封装
- **NFR3**：预留多审查者扩展和规则扩展点
- **NFR4**：终端命令白名单 + 工作目录沙箱 + 不记录密钥到日志
- **NFR5**：结构化日志 + 审查迭代计数 + 问题追踪
- **NFR6**：Python 3.11+，跨平台（Windows/Linux/macOS）
- **NFR7**：.env + AppConfig 统一配置管理
- **NFR8**：遵循 PEP 8，关键逻辑中文注释，公共 API 完整 docstring
- **NFR9**：单用户单实例模式，thread_id 隔离 checkpointer

## 约束

- **技术约束**：Python 3.11+，LangGraph 0.2.x，Node.js 18+
- **业务约束**：v1.0 不提供 Web UI，纯 CLI 交互
- **依赖约束**：LLM API（OpenAI 或 Anthropic），MCP 服务

## 假设

- 用户已安装 Node.js 18+（否则 MCP 服务自动降级）
- 用户已配置有效 LLM API 密钥
- 输入文档为 Markdown 格式（或可转换为 Markdown）

## 验收标准

### AC1：规格文档生成
- **给定**：用户通过 CLI 提供任务描述或文档
- **当**：系统初始化完成
- **则**：自动生成结构化规格文档，含概述、功能需求、验收标准等章节
- **验证**：programmatic
- **覆盖**：FR-S1, FR-S3

### AC2：六步审查输出
- **给定**：存在待审查规格文档
- **当**：DocReview 子智能体执行审查
- **则**：输出包含六步分析的结构化审查报告
- **验证**：programmatic
- **覆盖**：FR-D1~D7

### AC3：审查报告格式
- **给定**：DocReview 已完成审查
- **当**：生成审查报告
- **则**：报告严格遵循 PROMPT.md 定义的格式
- **验证**：human-judgment
- **覆盖**：FR-D7

### AC4：自动修订循环
- **给定**：审查结论为 Fail
- **当**：评估审查结果
- **则**：Supervisor 自动修订规格并重新提交审查
- **验证**：programmatic
- **覆盖**：FR-S5, FR-S6

### AC5：用户确认门禁
- **给定**：审查结论为 Pass 或 Conditional Pass
- **当**：路由到用户确认阶段
- **则**：系统中断并提示用户确认
- **验证**：human-judgment
- **覆盖**：FR-S7

### AC6：任务执行
- **给定**：用户已确认执行
- **当**：恢复工作流
- **则**：开始实际任务执行
- **验证**：programmatic
- **覆盖**：FR-S8

### AC7：用户拒绝处理
- **给定**：用户拒绝执行
- **当**：恢复工作流
- **则**：安全终止并保留所有状态
- **验证**：programmatic
- **覆盖**：FR-S7

### AC8：阅读工具功能
- **给定**：有测试文件存在
- **当**：调用阅读工具各项功能
- **则**：正确读取文件、搜索内容、列出目录
- **验证**：programmatic
- **覆盖**：FR-R1, FR-R2, FR-R3

### AC9：终端工具功能
- **给定**：有可执行命令
- **当**：调用终端工具执行
- **则**：返回完整 stdout/stderr/exit_code
- **验证**：programmatic
- **覆盖**：FR-T1, FR-T2, FR-T3

### AC10：联网搜索功能
- **给定**：有网络连接
- **当**：调用搜索工具
- **则**：返回相关网页结果
- **验证**：programmatic
- **覆盖**：FR-W1, FR-W2

### AC11：Sequential Thinking MCP
- **给定**：MCP 服务已启动
- **当**：调用多步推理
- **则**：正常返回推理结果
- **验证**：programmatic
- **覆盖**：FR-M1

### AC12：Context7 MCP
- **给定**：MCP 服务已启动
- **当**：解析库标识符或查询文档
- **则**：正常返回结果
- **验证**：programmatic
- **覆盖**：FR-M2

### AC13：审查历史持久化
- **给定**：完成多轮审查
- **当**：流程结束
- **则**：完整记录到 SQLite，支持后续对比
- **验证**：programmatic
- **覆盖**：FR-S9, FR-D8

### AC14：异常处理
- **给定**：LLM 超时或工具调用失败
- **当**：触发异常
- **则**：有合理错误处理和重试
- **验证**：programmatic
- **覆盖**：NFR1

### AC15：最大迭代次数限制
- **给定**：达到配置的最大迭代次数
- **当**：评估下一轮路由
- **则**：强制终止并给出明确提示
- **验证**：programmatic
- **覆盖**：FR-S6

### AC16：任务规划
- **给定**：用户提供任务描述
- **当**：Supervisor 处理
- **则**：分解为可跟踪的子任务清单
- **验证**：programmatic
- **覆盖**：FR-S2

### AC17：状态传递完整性
- **给定**：规格文档已生成
- **当**：提交给 DocReview
- **则**：所有必要状态完整传递
- **验证**：programmatic
- **覆盖**：FR-S4

### AC18：终端超时处理
- **给定**：命令执行超过配置超时
- **当**：超时发生
- **则**：进程被终止并返回超时状态
- **验证**：programmatic
- **覆盖**：FR-T4

### AC19：终端重试
- **给定**：命令执行失败
- **当**：触发重试机制
- **则**：自动重试（指数退避，最多 3 次）
- **验证**：programmatic
- **覆盖**：FR-T5

### AC20：危险命令拦截
- **给定**：尝试执行危险命令（如 rm -rf /）
- **当**：终端工具接收命令
- **则**：被白名单拦截
- **验证**：programmatic
- **覆盖**：FR-T6

### AC21：URL 内容获取
- **给定**：提供有效 URL
- **当**：调用 fetch_url
- **则**：获取内容并转为 Markdown
- **验证**：programmatic
- **覆盖**：FR-W2

### AC22：API 验证
- **给定**：提供第三方 API 规格
- **当**：调用验证
- **则**：返回可用性状态
- **验证**：programmatic
- **覆盖**：FR-W3

### AC23：依赖版本检测
- **给定**：项目存在依赖配置
- **当**：调用检测
- **则**：返回过时依赖列表
- **验证**：programmatic
- **覆盖**：FR-W4

### AC24：文档版本对比
- **给定**：有两个版本的文档
- **当**：调用 compare_versions
- **则**：生成 unified diff
- **验证**：programmatic
- **覆盖**：FR-R4

### AC25：路径安全
- **给定**：尝试访问工作目录外文件
- **当**：调用 read_file
- **则**：被拒绝访问
- **验证**：programmatic
- **覆盖**：FR-R5

### AC26：MCP 协议通信
- **给定**：MCP 服务运行中
- **当**：发送 MCP 请求
- **则**：JSON-RPC 协议通信正常
- **验证**：programmatic
- **覆盖**：FR-M3

### AC27：修订对比
- **给定**：有修订后的文档
- **当**：重新审查
- **则**：输出包含修订对比信息
- **验证**：human-judgment
- **覆盖**：FR-D8

### AC28：审查独立性
- **给定**：DocReview 执行审查
- **当**：生成报告
- **则**：仅包含问题和建议，不完整重写文档（Jaccard 相似度 ≤ 30%）
- **验证**：programmatic
- **覆盖**：FR-D9

### AC29：审查结论判定
- **给定**：审查完成
- **当**：判定结论
- **则**：严格执行规则：仅 Medium/Low → Pass，High 但无 Blocking → Conditional Pass，Blocking 或 AC 不完整 → Fail
- **验证**：programmatic
- **覆盖**：FR-D7
